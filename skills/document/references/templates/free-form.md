# Free-Form Template

Use this template for documentation that doesn't fit into the other categories.

## When to Use

- Documenting something that doesn't fit other templates
- Quick notes or observations
- Integration guides
- Miscellaneous technical documentation

## Template Structure

```markdown
# {document_title}

## Metadata

- **Timestamp**: {YYYY-MM-DD HH:MM:SS}
- **Project**: {project_name}
- **Category**: Free-Form
- **Tags**: {comma_separated_tags}
- **Related Commit**: {commit_hash if available}

## Content

(User provides structure and content freely)

## Files Referenced

(If applicable)

- `path/to/file1.ext` - Description
- `path/to/file2.ext` - Description

## Additional Context

(If applicable)
Any additional relevant context.

## References

(If applicable)

- Reference 1
- Reference 2
```

## Field Descriptions

### Metadata
- **Timestamp**: Auto-generated current date/time
- **Project**: Auto-detected from directory or git repo
- **Category**: Always "Free-Form"
- **Tags**: Relevant tags for search
- **Related Commit**: Related git commit if available

### Content
Free-form section where user provides their own structure and content. This is the main body of the document.

### Files Referenced
Optional list of files mentioned or relevant to this document.

### Additional Context
Optional section for any supporting information.

### References
Optional list of related resources or documentation.

## Notes

This template is intentionally minimal to allow maximum flexibility. Use it when:
- The content doesn't fit other templates
- You need a quick way to capture information
- The document has a unique structure
- You're documenting something experimental

If you find yourself using this template frequently for the same type of content, consider proposing a new specialized template.
